# Predicting Post-Disaster Public Assistance Funding
### Artificial Intelligence II — Final Project

---

## Project Overview

Following a federally declared disaster, FEMA disburses Public Assistance (PA) grants to state and local governments to rebuild public infrastructure. Determining how much funding a community needs is currently a slow, manual process relying on physical site inspections and bureaucratic damage assessments — a process that can take months.

This project builds a **Supervised Machine Learning Regression system** that takes high-level disaster characteristics as input and outputs a predicted dollar amount for the reconstruction grant — replacing a months-long accounting process with an instant, data-driven estimate.

**Two prediction levels are modeled:**
- **Level 1 — Strategic (per disaster):** Predicts total PA spending for an entire disaster event. Used by FEMA HQ for budget forecasting before boots are on the ground.
- **Level 2 — Operational (per project):** Predicts the grant amount for an individual reconstruction project. Used by local recovery coordinators to sanity-check individual claims.

## Features

### Engineered from FEMA data (no external join needed)
| Feature | Description |
|---|---|
| `declaration_lag_days` | Days between incident start and federal declaration |
| `incident_duration_days` | Length of the disaster event in days |
| `incident_season` | Season of incident (Winter / Spring / Summer / Fall) |
| `prior_disasters_5yr` | Count of prior disasters in same state in last 5 years |
| `n_counties_affected` | Number of counties designated per disaster (disaster level only) |

### Joined from Census (county level)
| Feature | Description |
|---|---|
| `population` | Total county population |
| `median_income` | Median household income ($) |
| `poverty_rate` | % of population below poverty line |

### Joined from FEMA NRI (county level)
| Feature | Description |
|---|---|
| `risk_score` | Overall county hazard risk score across 18 hazard types |

### Categorical features (encoded)
| Feature | Description |
|---|---|
| `incidentType` | Hurricane, Flood, Fire, Tornado, etc. |
| `stateCode` | 2-letter state abbreviation |
| `damageCategory` | A=Debris, B=Protective, C=Roads, D=Water, E=Buildings, F=Utilities, G=Parks |

---

## Target Variable

**`log_federal_share`** — the log1p-transformed value of `federalShareObligated` (total federal grant amount in $).

Log-transformation is applied because the raw dollar distribution is extremely right-skewed (range: $0 to ~$1B+). All models train and predict on the log scale; predictions are inverse-transformed (`expm1`) for presentation.

> ⚠️ **Dropped due to leakage:** `projectSize`, `projectAmount`, `totalObligated` — these are derived from or directly include the label value and must not be used as features.

---

## Models

All models are trained using a **time-based train/test split** (train: before 2018, test: 2018 and after) to simulate realistic forecasting conditions and prevent temporal leakage.

| Model | Purpose |
|---|---|
| `DummyRegressor` (mean) | Baseline — what we must beat |
| `Ridge Regression` | Linear baseline |
| `Random Forest` | Strong non-linear baseline, interpretable feature importance |
| `XGBoost / LightGBM` | Expected best performer |

**Evaluation metrics:** MAE, RMSE, R², MAPE

## Key Design Decisions

| Decision | Rationale |
|---|---|
| Two-level modeling | Disaster-level = budget forecasting (FEMA HQ). Project-level = operational triage (local agencies). Different stakeholders, different error tolerances. |
| Time-based split | Disasters are temporal. Random split would leak future data into training — unrealistic for forecasting. |
| Log-transform target | Raw `federalShareObligated` is heavily right-skewed. Log-scale stabilizes variance and improves model fit. |
| Census 5-Year ACS | More reliable than 1-year estimates for small/rural counties — exactly the counties most affected by disasters. |
| FEMA NRI over live weather API | Historical hazard risk scores are the correct substitute for a live weather API when training on historical events. |

## Limitations & Future Work

- Model cannot reliably predict once-in-a-generation events (Katrina, Harvey) — extreme outliers with no comparable training examples
- FEMA PA data skews toward states with more frequent disaster declarations — geographic bias in training data
- **Future:** NOAA Storm Events integration (wind speed, precipitation) for richer physical damage signal
- **Future:** Satellite-derived damage indices as features
- **Future:** Real-time API deployment for live disaster triage
